s_j(a,b) = tr(sigma_j Q_{a,b})
c_{a,b} = tr(Q_{a,b}) / N
p_{a,b}(theta) = c_{a,b} + 0.5 * s(a,b)^T theta

MLE(theta): minimize  -sum_{a,b} N_{a,b} * log(p_{a,b}(theta) + eps_log)
subject to:
  rho(theta) = I/N + 0.5 * sum_j theta_j sigma_j
  rho(theta) is PSD (via real-embedded PSD variable)
  p_{a,b}(theta) >= 0
  rho(theta) - eta I is PSD
  eta is promoted to at least 1e-3 and must satisfy eta < 1/N

Post-solve safeguard:
  if status is optimal/optimal_inaccurate and min_eig(rho_hat) < eta,
  project rho_hat to enforce the floor and map back to theta_hat

Used by simulation:
eta_mle = 1e-3, eps_log = 1e-12, solver = SCS

3) What Was the Mistake?

  
Mistake made: treating one trajectory (n_rep = 1) as sufficient evidence for convergence quality at n = 400. That is too noisy and can make exact/GI1 look inconsistent relative to oracle.

  
There is also a systematic finite-sample effect from the implemented policy structure: it is certainty-equivalent greedy (no forced exploration term), and L2 initialization uses only settings 1:3 initially (/Users/cyberslave/GitHub/quantum_tomography_2026/R_implementations/07_adaptive_design.R:95-101), not all nine axes.
