1. Goal and ratio definition

For each case (system, library, loss), we compute \[ \mathrm{Ratio}(\alpha)= \frac{\mathrm{tr}\!\left(G(\theta_{\mathrm{true}})\, I_{\mathrm{unif}}(\theta_{\mathrm{true}})^{-1}\right)} {\min_{\pi\in\Delta_k}\;\mathrm{tr}\!\left(G(\theta_{\mathrm{true}})\, I_{\pi}(\theta_{\mathrm{true}})^{-1}\right)}. \] Here \(I_\pi(\theta)=\sum_{a=1}^k\pi_a I_a(\theta)\) and \(\Delta_k\) is the simplex.

2. State parametrization and Bloch coordinates

2.1 One qubit

The one‑qubit Pauli basis satisfies \(\mathrm{tr}(\sigma_j\sigma_k)=2\delta_{jk}\). The state model is \[ \rho(\theta)=\frac{I_2}{2}+\frac12\sum_{j=1}^3\theta_j\sigma_j, \quad \theta_j=\mathrm{tr}(\rho\sigma_j). \]

2.2 Two qubits

The two‑qubit basis uses \[ \sigma_{(\alpha,\beta)}=\frac{1}{\sqrt{2}}(\sigma_\alpha\otimes\sigma_\beta), \quad (\alpha,\beta)\ne(I,I), \] so \(\mathrm{tr}(\sigma_j\sigma_k)=2\delta_{jk}\). The state model is \[ \rho(\theta)=\frac{I_4}{4}+\frac12\sum_{j=1}^{15}\theta_j\sigma_j. \] With this scaling, \[ \theta_j=\mathrm{tr}(\rho\sigma_j). \] If \(P=\sigma_\alpha\otimes\sigma_\beta\) is an unscaled Pauli product, then \(\sigma_j=P/\sqrt{2}\) and \(\theta_j=\tfrac{1}{\sqrt{2}}\mathrm{tr}(\rho P)\).

3. True state used in the table

The table uses a fixed pure state mixed with the maximally mixed state: \[ \rho_{\mathrm{true}}=\alpha\,\rho_{\mathrm{pure}}+(1-\alpha)\frac{I_N}{N}. \]

One‑qubit pure state: \(|+\rangle\langle+|\) where \(|+\rangle=(|0\rangle+|1\rangle)/\sqrt{2}\).
Two‑qubit pure state: \(|00\rangle\langle 00|\).

4. Measurement libraries

L1 (1q): Pauli PVMs (X,Y,Z).

L2 (1q): Nine-axis PVMs.

L3 (1q): Four random axes (seed 42).

A (2q): 15 Pauli‑parity PVMs \(Q_{P,\pm}=\tfrac12(I_4\pm P)\).

B (2q): 5 mutually unbiased bases (each a 4‑outcome rank‑1 PVM).

5. Affine probability model

The code builds an affine model \[ p_{a,b}(\theta)=c_{a,b}+\frac12 s_{a,b}^\top\theta, \] where \[ c_{a,b}=\frac{\mathrm{tr}(Q_{a,b})}{N}, \quad s_{a,b,j}=\mathrm{tr}(\sigma_j Q_{a,b}). \]

6. Fisher information per setting

For each setting \(a\), the Fisher matrix is \[ [I_a(\theta)]_{jk}=\frac14\sum_{b=1}^{r_a}\frac{s_{a,b,j}\,s_{a,b,k}}{p_{a,b}(\theta)}. \]

7. Loss metrics

Frobenius: \[ G_F=\frac14[\mathrm{tr}(\sigma_j\sigma_k)]_{jk}=\frac12 I_d. \]

Bures: \[ [G_B(\theta)]_{jk}=\frac{1}{16}\mathrm{tr}\!\left(\sigma_j\,\Omega_\rho^{-1}(\sigma_k)\right), \quad \Omega_\rho(X)=\tfrac12(\rho X + X\rho). \]

Observable: \[ G_O=\frac14\sum_\ell o_\ell o_\ell^\top, \quad [o_\ell]_k=\mathrm{tr}(\sigma_k O_\ell). \]

8. Oracle vs uniform risk

The proxy risk is \[ R(\pi)=\mathrm{tr}\!\left(G(\theta_{\mathrm{true}})\,I_\pi(\theta_{\mathrm{true}})^{-1}\right). \]

Uniform design uses \(\pi_{\mathrm{unif}}=(1/k,\dots,1/k)\). The ratio is \(R(\pi_{\mathrm{unif}})/R(\pi^*)\).

9. Oracle design optimization

Given the per‑setting Fisher matrices \(I_a(\theta_{\mathrm{true}})\), define \[ I_\pi(\theta_{\mathrm{true}})=\sum_{a=1}^k \pi_a I_a(\theta_{\mathrm{true}}), \qquad \pi\in\Delta_k. \] The oracle design minimizes \(R(\pi)=\mathrm{tr}(G I_\pi^{-1})\). In the code, this is solved via the standard SDP relaxation: \[ \min_{\pi,W}\ \mathrm{tr}(GW)\quad \text{s.t.}\quad \sum_a\pi_a I_a = I_\pi,\ \pi\ge0,\ \sum_a\pi_a=1,\ \begin{pmatrix} I_\pi & I \\ I & W\end{pmatrix}\succeq 0. \]

By the Schur complement, the block constraint enforces \(W \succeq I_\pi^{-1}\), so at the optimum \(W=I_\pi^{-1}\) (when \(I_\pi\) is positive definite), and the objective equals the desired oracle risk.

After solving for \(\pi^*\), the code recomputes \(I_{\pi^*}=\sum_a \pi_a^* I_a\) and evaluates \[ R_{\mathrm{oracle}}=\mathrm{tr}\!\left(G\,I_{\pi^*}^{-1}\right) \] using proxy_risk() (which uses a pseudo‑inverse if needed).

If the SDP returns a numerically poor solution (e.g., \(R_{\mathrm{oracle}} > R_{\mathrm{unif}}\)), the code falls back to a deterministic mirror‑descent solver on the simplex. The update uses the exact gradient \[ g_a(\pi)=-\mathrm{tr}\!\left(G\,I_\pi^{-1} I_a\, I_\pi^{-1}\right), \] then applies an exponentiated‑gradient step \[ \pi_a \leftarrow \frac{\pi_a \exp(-\eta g_a)}{\sum_{b}\pi_b \exp(-\eta g_b)} \] with backtracking to ensure monotone improvement. This guarantees the oracle risk is never worse than the uniform design.

10. End‑to‑end algorithm

For each case:

1) Build library \(\{Q_{a,b}\}\).

2) Build basis \(\{\sigma_j\}\).

3) Form \(\rho_{\mathrm{true}}\) from the fixed pure state and \(\alpha\).

4) Compute \(\theta_{\mathrm{true}}\).

5) Build \(S_{a,b}\), \(c_{a,b}\).

6) Compute \(I_a(\theta_{\mathrm{true}})\) for all settings.

7) Build \(G(\theta_{\mathrm{true}})\).

8) Compute uniform risk.

9) Solve oracle design, compute oracle risk.

10) Output ratio = uniform / oracle.
