R_final_implementations: Numbered Spec Logic Summary

This page summarizes each numbered markdown file and the corresponding script logic.

Mathematical Core

State model: \(\rho(\theta)=I/N+\frac12\sum_j \theta_j\sigma_j\).

Proxy risk: \(R_n(G)=\mathrm{tr}(G J_n^{-1})\), where \(J_n=\sum_a N_a I_a\).

Multinomial adaptive sampling: each adaptive step draws \(\mathrm{Multinomial}(n_{trials}, p_a)\).

Baseline policy now uses permutation-cyclic setting selection (policy key remains uniform).

PairPrimary logic
00_simulation_parameters.*Central parameter source, profile defaults, CLI/config merges, baseline-mode normalization.
01_utilities.*Common helper functions for complex matrices, seeds, and utility operations.
02_state_basis.*Basis builders and conversions between density matrices and Bloch coordinates.
03_measurement_library_1q.*One-qubit measurement libraries L1/L2/L3 and associated indexing metadata.
04_measurement_library_2q.*Two-qubit measurement libraries A/B and unified library objects.
05_mle_cvxr_solver.*MLE estimation via CVX/PGD with density-floor control and solver diagnostics.
06_fisher_and_metrics.*Fisher information, local metrics (Frobenius/Bures/observable), and risk proxies.
07_run_caseI_caseII_relative_oracle_multinomial.*Main all-15 alpha runner, trajectory logging, oracle-limit comparisons, and plot generation.
08_verify_caseI_caseII_outputs.*Post-run checks: \(\mathbb{E}[D(\hat{\rho},\rho)^2] \approx \frac{1}{n}\mathrm{tr}(G_D(\theta)\,I(\theta)^{-1})\) gap tables, plot existence, ranking checks, trajectory coverage.
09_replot_caseI_caseII_xranges.*Replot from saved RDS with standardized x-range windows and summary panel regeneration.
10_run_case11_bures_rep1000_parallel.*Dedicated case11 parallel runner and case11-specific proxy/relative/Bures plots.
11_build_alpha_plus_summary_pdf.*Build compact 3-page alpha report PDF from three summary PNG files.
12_smoke_test_alpha0p9_all15.*Fast end-to-end test orchestrator for run + PDF build + verification.

Execution Flow

  1. Load config from 00_simulation_parameters.R.
  2. Run simulation (07 or 10).
  3. Generate compact PDF via 11.
  4. Validate output structure via 08.
  5. Use 12 for fast smoke validation.
